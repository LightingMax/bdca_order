# 多文件上传功能说明

## 功能概述

现在系统支持**连续上传多个ZIP文件**，而不需要刷新页面。用户可以：
1. 拖拽第一个ZIP文件 → 处理完成，显示结果
2. 拖拽第二个ZIP文件 → 结果追加到第一个文件的结果后面
3. 拖拽第三个ZIP文件 → 结果继续追加
4. 最终看到所有文件的累计处理结果

## 主要改进

### 1. 累积显示结果
- **之前**：每次上传新文件，之前的结果被清空
- **现在**：新文件的结果追加到现有结果后面
- **优势**：支持批量报销，一次处理多个文件

### 2. 智能结果分组
- 每次上传的文件结果都有独立的分组标题
- 显示本次处理的订单数量和累计总数
- 便于区分不同批次的上传结果

### 3. 累计统计信息
- 显示本次处理的订单数量和金额
- 显示累计处理的订单总数和总金额
- 让用户清楚了解整体报销情况

## 使用场景

### 场景1：单次报销多个文件
```
用户有3个ZIP文件需要报销：
1. 拖拽"阳光出行-32.13元-3个行程.zip" → 显示3个订单
2. 拖拽"高德打车-94.42元-2个行程.zip" → 追加2个订单，累计5个订单
3. 拖拽"T3-77.06-1.zip" → 追加1个订单，累计6个订单
最终：一次性看到所有6个订单，总金额¥203.61
```

### 场景2：分批次上传
```
用户先上传部分文件：
1. 拖拽"滴滴出行-28.50元-1个行程.zip" → 显示1个订单
2. 拖拽"美团打车-15.80元-1个行程.zip" → 追加1个订单，累计2个订单

稍后继续上传：
3. 拖拽"曹操出行-45.20元-2个行程.zip" → 追加2个订单，累计4个订单
最终：看到所有4个订单的完整信息
```

## 技术实现

### 1. 结果累积逻辑
```javascript
// 将新结果添加到现有结果中，而不是替换
if (typeof processedFiles === 'undefined') {
    processedFiles = [];
}
processedFiles = processedFiles.concat(response.results);
```

### 2. 智能分组显示
```javascript
// 检查是否已有订单结果区域
let ordersDiv = document.getElementById('orders-section');
if (!ordersDiv) {
    // 如果不存在，创建新的订单结果区域
    ordersDiv = document.createElement('div');
    ordersDiv.id = 'orders-section';
    // ... 创建标题
}
```

### 3. 累计统计计算
```javascript
// 显示本次处理结果统计
const newFilesCount = response.results.length;
const newAmount = response.total_amount || response.results.reduce((sum, result) => sum + result.amount, 0);

// 计算累计统计
const totalFiles = processedFiles.length;
const totalAmount = processedFiles.reduce((sum, result) => sum + result.amount, 0);
```

## 操作流程

### 1. 连续上传文件
1. 拖拽第一个ZIP文件到上传区域
2. 等待处理完成，查看结果
3. 继续拖拽第二个ZIP文件
4. 新结果自动追加到现有结果后面
5. 重复步骤3-4，直到所有文件处理完成

### 2. 管理结果
- **查看**：所有结果按上传批次分组显示
- **打印**：可以单独打印某个订单，或批量打印全部
- **清空**：点击"清空结果"按钮清空所有结果

### 3. 注意事项
- 页面刷新会清空所有结果
- 关闭浏览器会丢失未保存的结果
- 建议处理完成后及时打印或保存

## 优势总结

1. **提升效率**：一次处理多个文件，无需重复操作
2. **结果累积**：所有文件结果集中显示，便于管理
3. **智能分组**：按上传批次分组，结构清晰
4. **累计统计**：实时显示总订单数和总金额
5. **操作灵活**：支持单独打印、批量打印、清空结果

这个功能大大改善了用户体验，特别适合需要报销多个文件的场景，让用户能够一次性看到所有报销订单的完整信息。
