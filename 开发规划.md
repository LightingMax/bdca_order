# 订单报销系统开发规划

## 项目概述

本项目旨在构建一个简单的网页系统，用于处理订单报销文件。系统主要功能包括：
1. 解压用户上传的ZIP文件
2. 识别并匹配订单相关的PDF和XML文件
3. 合并PDF文件（行程单在前，发票在后）
4. 调整合并后的PDF为适合打印的格式
5. 自动调用打印机打印输出
6. 记录用户打印信息并展示统计数据

## 技术栈选择

- **前端**：HTML, CSS, JavaScript, Vue.js
- **后端**：Python + Flask
- **PDF处理**：PyPDF2 或 pikepdf
- **ZIP处理**：zipfile 库
- **XML解析**：xml.etree.ElementTree
- **打印功能**：cups 或 win32print (根据部署环境)
- **数据存储**：JSON文件

## 功能模块详细设计

### 1. 文件上传与解压模块
- 接收用户上传的ZIP文件
- 解压到临时目录
- 识别所有PDF和XML文件

### 2. 文件匹配与分析模块
- 解析XML文件获取订单信息（金额等）
- 根据文件名和内容识别行程单PDF和发票PDF
- 将属于同一订单的文件进行匹配

### 3. PDF处理模块
- 合并同一订单的行程单和发票PDF
- 调整PDF内容为适合打印的格式（一页显示）
- 生成最终的PDF文件

### 4. 打印模块
- 检测系统打印机
- 发送合并后的PDF到打印机
- 处理打印状态和错误

### 5. 用户信息记录模块
- 获取用户MAC地址
- 记录用户打印信息（时间、文件数量、金额等）
- 将信息保存到本地JSON文件

### 6. 统计展示模块
- 读取本地JSON文件中的用户信息
- 计算各用户累计打印的发票金额
- 提供API接口供前端展示

## 风险评估与解决方案

1. **PDF处理兼容性问题**
   - 风险：不同来源的PDF格式可能不同，导致处理失败
   - 解决方案：增加PDF格式检测，采用多种PDF库组合处理

2. **打印机兼容性问题**
   - 风险：不同操作系统和打印机型号的兼容性
   - 解决方案：实现多平台打印接口，提供打印预览和手动打印选项

3. **MAC地址获取限制**
   - 风险：浏览器环境可能限制获取MAC地址
   - 解决方案：考虑使用其他标识方式，如浏览器指纹或IP+用户代理组合

4. **大文件处理性能**
   - 风险：大量或大体积PDF处理可能导致性能问题
   - 解决方案：实现异步处理，添加进度反馈，优化内存使用
